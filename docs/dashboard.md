# Dashboard Notes

The `DavidSan` dashboard is a key part of the larger product vision.

It is designed around one main idea:

The assistant should not only answer. It should help the user stay focused on the most important work.

## What The Dashboard Shows

The dashboard is intended to make decision support visible in one place:

- the top current priority
- why it is ranked first
- how well it matches the user's goals and profile
- supporting references and evidence
- the next recommended action
- the active agent conversation alongside execution context

## Why The Timeline Matters

The timeline is one of the most important UI components in the system.

Its job is to prevent important work from disappearing into a backlog or chat history.

It helps by:

- making deadlines visible
- showing sequencing constraints
- surfacing what should happen next
- keeping the user anchored on high-priority execution
- reducing reactive drift

In other words, the timeline turns intent into momentum.

## Product Meaning

This is important for the public story of the project because it shows that `DavidSan` is not just a text generation layer.

It is being designed as a personal AI workflow that combines:

- understanding
- prioritization
- reasoning
- execution support
- and focus management
