# Demo Walkthrough

This short walkthrough explains what the public dashboard demo is trying to show.

## 1. Priority-first interface

The left panel is not a generic task list.

It highlights the single most important current opportunity, why it ranks first, and how well it fits the user's goals and profile.

## 2. Reasoning with evidence

The recommendation includes:

- a fit score
- rationale for prioritization
- supporting references
- a breakdown of profile, timing, ROI, and risk

The goal is to make the recommendation interpretable instead of opaque.

## 3. Proactive agent behavior

The right panel shows a `Phi-4`-based agent that does more than answer.

It identifies missing context, asks direct questions when needed, and pushes the user toward the next meaningful action.

## 4. Timeline as focus control

The timeline is one of the most important parts of the demo.

It keeps attention on the highest-priority execution path, makes sequencing visible, and reduces the chance that strategic work gets lost in reactive noise.

## 5. Why this matters

The public demo is meant to show that `DavidSan` is not only about language generation.

It is about combining understanding, prioritization, execution support, and focus management in a privacy-first workflow.
